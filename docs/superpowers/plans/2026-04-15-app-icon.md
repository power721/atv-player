# App Icon Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Load a packaged `app.svg` at application startup so source-run top-level Qt windows inherit the same app icon.

**Architecture:** Keep the icon asset inside `src/atv_player/icons/` with the existing SVG controls, add a small path helper in `src/atv_player/app.py`, and set the icon once on `QApplication` during `build_application()`. The current wheel build already packages SVG files under `src/atv_player/icons/`, so implementation only needs to move the asset into that directory and verify the built wheel contains it.

**Tech Stack:** Python 3.12, PySide6, pytest, hatchling/uv build

Update `src/atv_player/app.py` to import `QIcon`, add a helper for the packaged asset path, and set the icon inside `build_application()`:

```python
from pathlib import Path

from PySide6.QtCore import QObject
from PySide6.QtGui import QIcon
from PySide6.QtWidgets import QApplication, QWidget


def _app_icon_path() -> Path:
    return Path(__file__).resolve().parent / "icons" / "app.svg"


def build_application() -> tuple[QApplication, SettingsRepository]:
    app = QApplication([])
    app.setApplicationName("atv-player")
    app.setWindowIcon(QIcon(str(_app_icon_path())))
    data_dir = Path.home() / ".local" / "share" / "atv-player"
    repo = SettingsRepository(data_dir / "app.db")
    return app, repo
```
