# Media Folder Breadcrumb Design

## Summary

`网络直播`、`Emby`、`Jellyfin` 页签在进入文件夹后，需要显示可点击的面包屑，并允许用户点击上级节点返回对应层级。

这三个页签当前都复用 `DoubanPage` 展示卡片列表，但文件夹导航状态只存在于 `MainWindow` 的一次性点击处理里。页面能继续向下进入文件夹，不能显示当前层级，也不能回到上级。

## Goals

- 为 `网络直播`、`Emby`、`Jellyfin` 提供一致的可点击面包屑导航。
- 进入文件夹时在当前页签内更新面包屑层级。
- 点击任意上级面包屑时重新加载对应层级内容。
- 切换左侧分类时重置文件夹导航，回到该分类根层。

## Non-Goals

- 不修改 `豆瓣电影` 与 `电报影视` 的交互。
- 不为文件浏览页重写已有面包屑实现。
- 不引入跨页签共享的导航历史。

## Design

### Shared Page State

在 `DoubanPage` 中新增一个可选的文件夹导航模式，默认关闭。开启后页面维护一组面包屑节点，每个节点至少包含：

- 展示名称
- 节点标识（分类 `type_id` 或文件夹 `vod_id`）
- 节点类型（首页、分类根层、文件夹）

页面负责渲染面包屑按钮，并在点击时发出一个“请求导航到某个节点”的信号；页面本身不直接调用控制器加载数据。

### Root and Folder Navigation

`MainWindow` 继续作为 `网络直播`、`Emby`、`Jellyfin` 的导航协调者：

- 分类切换后，把面包屑重置为 `首页 / 当前分类`
- 文件夹点击后，调用各自控制器的 `load_folder_items(vod_id)`，成功后把该文件夹追加到面包屑
- 面包屑点击后：
  - 点击 `首页` 或当前分类根层时，使用 `load_items(selected_category_id, 1)` 回到分类根层
  - 点击某个中间文件夹时，使用 `load_folder_items(vod_id)` 回到该文件夹，并截断其后的层级

### Error Handling

- 如果文件夹加载失败，保留当前列表和当前面包屑，不前进也不回退。
- 如果面包屑回退加载失败，保留当前列表和当前面包屑，并继续通过现有错误弹窗反馈。

## Testing

- `DoubanPage`：
  - 开启导航模式时渲染 `首页 / 分类 / 文件夹...` 面包屑
  - 点击面包屑发出目标节点信号
  - 切换分类时重置到分类根层
- `MainWindow`：
  - `网络直播` 文件夹进入后更新面包屑
  - `Emby`、`Jellyfin` 文件夹进入后更新面包屑
  - 点击面包屑能够重新加载上级层级而不是打开播放器
